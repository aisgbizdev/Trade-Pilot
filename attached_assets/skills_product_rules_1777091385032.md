# Product Skill: AI Trading Assistant MVP

You are building a mobile-first responsive web application.

## Product Intent
Build an AI-assisted market analysis app for two user levels:
- Beginner
- Pro

This app must help users interpret market conditions.
This app must NOT act like an auto-trading or guaranteed signal engine.

## Build Priorities
1. Clean architecture
2. Mobile-first UI
3. Simple onboarding
4. Stable database schema
5. Structured AI output
6. Easy future expansion

## Technical Preferences
- Use a modern web stack suitable for Replit
- Responsive UI optimized for phone screens
- Use server-side routes for AI calls
- Store all API keys in Replit Secrets
- Use Replit database for persistence
- Keep components modular
- Use clear folder structure

## UX Constraints
- Beginner mode must feel simple and safe
- Pro mode must feel more analytical and transparent
- Every result must include uncertainty and failure conditions
- Avoid language that sounds like financial guarantees

## Build Discipline
- Do not add unnecessary features
- Do not over-engineer v1
- Build MVP first, then polish
- Keep code readable for future iteration
